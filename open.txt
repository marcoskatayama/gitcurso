3 - Quais arquivos versionar?
Ao criar o repositorio, precisa informar ao git quais arquivos devem ser versionados
Até o momento informamos que tem um repositorio na pasta Temp

O comando git status mostra o status do arquivo. Se adicionar um arquivo na pasta e digitar git status, ira mostrar os arquivos com o status Untracked, ou seja, não é rastreavel.

Para rastrear
git add nome do arquivo(ou apertar o tab que completa o nome)

se verificar agora com git status, agora ele esta como rastreado.

Se tiver varios arquivos para rastrear, melhor utilizar 
git add --all ou git add -A ou git add .

7 - Desfazer
Caso tenha feito alguma alteração e deseja desfazer, só ditigar git checkout e o nome do arquivo.

Agora se tem varios arquivos que deseja voltar como eram antes pode usar
git reset --hard

Porem se existirem arquivos que ainda não foram rastreados, o git reset não vai remover estes arquivos, precisara usar
git clean -f
 
8 - Ignorar arquivos
No projeto alguns arquivos são criados pelas frameworks em seu computador local por exemplo com arquivos de configuação, formatação, preferencia, dll que não seriam uteis para outros usuários, portanto é possivel criar um arquivo no git para ignorar esste arquivos.
Basta abrir o CMD do Windows, criar um arquivo qualquer por exemplo a.txt, depois no cmd renomear para .gitignore
ren a.txt .gitignore
precisa ser dentro do CMD pois direto na pasta o windows não permite criar um arquivo sem nome.
No site do github existem varias listas de .gitignore

15 - Branch
É uma ramificação para que o projeto original siga varios caminhos. Por exemplo, tenho um projeto e quero fazer modificações sem alterar o original, no final posso fazer um merge e unir estes projetos.

git checkout e o nome do branch que deseja ir

se existir arquivos não commitados o git não vai permitir alterar de branch, para isso precisa git reset --hard ou commitar

Para listar as branch git branch

Atalho em vez de usar git branch e depois git checkout pode usar git checkout -b nome da branch

16 - Enviar branch para o servidor
git push --set-upstream origin nome do branch

atalho
git push -u origin nome do branch

17 - Atualizando branch
git pull
depois é só fazer git checkout nome da branch nova que vai aparecer a nova branch que esta no servidor

18 - Excluir branch local
git branch -d nome da branch
git branch -D nome da branch *comando para forçar a remoção

19 - Excluir branch do repositorio
git push --delete origin nome da branch

20 - Renomear branch local
Precisa estar na branch que quer renomear
git branch -m novo nome

Se não estiver na branch
git branch -m nome da branch que quer renomear e o novo nome

20 - Renomear branch no repositorio
Não é possivel, o jeito seria copiar a versão mais atual da branch fazendo o git pull, depois localmente alterar o nome, deletar a branch atual do servidor e mandar a nova branch para o servidor. Neste exemplo no servidor a branch atual é task1 e nome que vamos alterar será task2
1º git pull (para pegar a versão mais atual do servidor)
2º git branch -m task2 (renomeando localmente a branch)
3º git push --delete origin task1 (deletar a branch no servidor)
4º git push -u origin task2 (enviar a nova branch para o servidor) 

21 - Mesclando alterações
Exemplo feito alteração na branch developer e queremos adicionar na branch original main, basta acessar a branch main (git checkout main) e digitar o comando
git marge developer

O mesmo já irá fazer o commit na main se não houver conflitos.
